# Домашнее задание по SQL

## Easy

### E1

В таблице `order_items` есть цена товара в момент покупки: `order_items.price`.  
В таблице `products` есть текущая цена товара: `products.price`.

Найдите строки заказов, где цена покупки отличается от текущей цены товара.

Выведите:

- `order_id`;
- `product_id`;
- `product_name`;
- цену из `order_items`;
- цену из `products`;
- разницу между ценами.

---

### E2

Найдите заказы, в которых суммарно купили больше 5 единиц товаров.

Суммарное количество считайте как сумму `quantity` по строкам заказа.

Выведите:

- `order_id`;
- количество товарных позиций;
- суммарное количество единиц товара.

---

### E3

Найдите товары, которые хотя бы раз были проданы дороже текущей цены из таблицы `products`.

Для каждого товара выведите:

- `product_id`;
- `product_name`;
- текущую цену;
- максимальную цену продажи;
- разницу между максимальной ценой продажи и текущей ценой.

---

### E4

Найдите пользователей, у которых есть заказ, созданный раньше даты регистрации пользователя.

Сравнивайте:

- `users.registered_at`;
- `orders.order_date`.

Выведите:

- `user_id`;
- `email`;
- дату регистрации;
- `order_id`;
- дату заказа.

---

### E5

Найдите заказы, в которых все товары относятся только к одной категории.

Выведите:

- `order_id`;
- категорию;
- количество разных товаров в заказе;
- сумму заказа по строкам `order_items`.

---

## Medium

### M1

Для каждого заказа посчитайте, какую долю от суммы заказа занимает самая дорогая товарная позиция.

Стоимость товарной позиции считайте как:

```sql
quantity * price
```

Выведите:

- `order_id`;
- общую сумму заказа по строкам;
- стоимость самой дорогой позиции;
- долю самой дорогой позиции в процентах.

---

### M2

Для каждого пользователя с минимум двумя оплаченными заказами сравните:

- среднюю сумму его оплаченных заказов;
- сумму его последнего оплаченного заказа.

Выведите пользователей, у которых последний оплаченный заказ больше их средней суммы оплаченного заказа.

Выведите:

- `user_id`;
- `email`;
- количество оплаченных заказов;
- среднюю сумму оплаченного заказа;
- сумму последнего оплаченного заказа;
- разницу между последним заказом и средним.

---

### M3

Для каждой категории посчитайте, в скольких оплаченных заказах она встречалась вместе хотя бы с одной другой категорией.

Выведите:

- категорию;
- количество таких заказов;
- общее количество оплаченных заказов с этой категорией;
- долю заказов, где категория была куплена вместе с другими категориями.

---

## Hard

### H1

Для каждого пользователя определите категорию, на которую он потратил больше всего денег в оплаченных заказах.

Выведите:

- `user_id`;
- `email`;
- категорию;
- выручку пользователя в этой категории;
- общую оплаченную выручку пользователя;
- долю категории в выручке пользователя.

Если у пользователя две категории с одинаковой максимальной выручкой, выведите обе.

---

### H2

Для каждого пользователя найдите последовательность его оплаченных заказов.

Для каждого заказа, начиная со второго, посчитайте:

- сумму текущего заказа;
- сумму предыдущего заказа;
- разницу между текущим и предыдущим заказом;
- процент изменения относительно предыдущего заказа.

Выведите только те строки, где изменение по модулю больше 20%.

---

### H3

Найдите оплаченные заказы, где одна товарная категория занимает больше 80% суммы заказа.

Для каждого такого заказа выведите:

- `order_id`;
- доминирующую категорию;
- сумму заказа;
- сумму по доминирующей категории;
- долю доминирующей категории в заказе.

---

### H4

Для каждого пользователя определите, когда он впервые купил товар из новой для себя категории.

Например, если пользователь сначала покупал только `Electronics`, а потом впервые купил `Accessories`, нужно вывести дату появления новой категории.

Выведите:

- `user_id`;
- `email`;
- категорию;
- дату первого заказа с этой категорией;
- порядковый номер новой категории для пользователя.

---

### H5

Найдите заказы, где сумма по строкам `order_items` отличается от `orders.amount`, и при этом отличие составляет больше 10% от `orders.amount`.

Выведите:

- `order_id`;
- `user_id`;
- статус заказа;
- сумму из `orders.amount`;
- сумму по строкам заказа;
- абсолютную разницу;
- процент отклонения.

---

## Bonus

### B1

Для каждого оплаченного заказа рассчитайте индекс разнообразия корзины:

```text
количество разных категорий / количество разных товаров
```

Выведите:

- `order_id`;
- количество разных товаров;
- количество разных категорий;
- индекс разнообразия.

Отсортируйте заказы по индексу разнообразия по убыванию.

---

### B2

Найдите пользователей, которые в оплаченных заказах покупали товары только из одной категории.

Выведите:

- `user_id`;
- `email`;
- категорию;
- количество оплаченных заказов;
- сумму покупок в этой категории.

---

### B3

Для каждого товара посчитайте, насколько сильно менялась цена продажи в `order_items`.

Выведите товары, у которых минимальная цена продажи отличается от максимальной.

Для каждого товара выведите:

- `product_id`;
- `product_name`;
- минимальную цену продажи;
- максимальную цену продажи;
- разницу;
- количество заказов, где товар встречался.
